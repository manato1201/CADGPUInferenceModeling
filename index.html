① Zバッファ深度画像による隠線判定

3DモデルをCAD図面に変換する際、「どのエッジが手前に見えるか」を正確に判定することが課題だった。 従来の法線ベース手法では曲面・自己遮蔽を正確に扱えず、99.9%が隠線に誤分類されていた。 GPU不使用・numpy のみでバリセントリック座標法のZバッファラスタライザを自前実装することで解決した。

# バリセントリック座標法でZバッファに三角形を描画
def _build_zbuffer(verts, faces, ax_h, ax_v, ax_d, size=512):
    z_buf = np.full((size, size), np.inf, dtype=np.float32)
    for face in faces:
        a, b, c = face
        # バウンディングボックス内の全ピクセルにバリセントリック座標を計算
        area = (h1-h0)*(v2-v0) - (h2-h0)*(v1-v0)
        w0 = ((h1-h2)*(gx-h2) + (v2-v1)*(gy-v2)) / area
        w1 = ((h2-h0)*(gx-h0) + (v0-v2)*(gy-v0)) / area
        w2 = 1.0 - w0 - w1
        # 三角形内部のピクセルのみ深度を更新
        inside = (w0 >= -0.01) & (w1 >= -0.01) & (w2 >= -0.01)
        depth  = w0*d0 + w1*d1 + w2*d2
        z_buf[gy[inside], gx[inside]] = np.minimum(
            z_buf[gy[inside], gx[inside]], depth[inside]
        )

② CP932エンコードのレイヤー名自動判定

日本のCADソフトで作成したDXFはレイヤー名がCP932（Shift_JIS）でエンコードされているが、 ezdxfはそれをcp1252のMojibakeとして読み込む。 この問題を解決するため、各キーワードをCP932→cp1252変換してMojibakeパターンとして保持し、 ezdxf内部の文字列と直接マッチングする手法を実装した。 

# ezdxfのlatin1 Mojibakeパターンを事前生成
def _cp932_moj(text: str) -> str:
    b = text.encode("cp932")
    result = []
    for byte in b:
        if 0x80 <= byte <= 0xFF:
            try:
                result.append(bytes([byte]).decode("cp1252"))
            except ValueError:
                result.append(chr(0xDC00 + byte))  # surrogate
        else:
            result.append(chr(byte))
    return "".join(result)

# 壁1階/壁2階を自動検出 → 異なるZ座標に積み上げ
_WALL_MOJ   = [_cp932_moj("壁")]   # "•Ç" にマッチ
_FLOOR1_MOJ = [_cp932_moj("1階")]  # Z = 0〜2500mm
_FLOOR2_MOJ = [_cp932_moj("2階")]  # Z = 3000〜5500mm

③ Union-Find による端点スナップ

建築平面図の壁線分は隣接する壁との接合点が微妙にずれていることが多く、 そのまま押し出すと壁に隙間・浮きが発生する。 Union-Find アルゴリズムで80mm以内の近接端点を統合することでこの問題を解消した。

④ 平行線ペア検出による壁厚自動推定

建築CADの壁は2本の平行線で表現されることが多い。 各線分の角度・中点・垂直距離を計算し、平行かつ80〜500mmの間隔にある線分ペアを検出して その距離の中央値を壁厚として採用する（実測: 100mm）。

04 — Development Process

開発の過程

v0.1
Zero123++ + TSDF Fusion パイプライン
6視点生成 → 深度推定 → TSDF Fusion でメッシュ化。カメラ姿勢とTSDFの不整合で花びら状のノイズが発生。建築物に対してZero123++の学習分布とのズレが大きく改善困難と判断。
v0.5
TripoSR への切り替え
TSDFを廃止しTripoSRで単視点→直接メッシュ変換へ。torchmcubesビルド問題・RGBA→RGB変換・has_vertex_color引数エラーなど複数の環境依存エラーを解決。形状は改善されたが建物特化ではないため限界あり。
v1.0
推論なし押し出しパイプラインの品質向上に方針転換
建築特化モデル（HouseCrafter等）は非公開・実用困難と調査で判明。「汎用推論より正確な押し出しの方がゲームアセットとして価値が高い」と判断し、方向Aへ切り替え。
v1.5
floor_plan_extruder v2: 5つの品質改善
① Union-Find端点スナップ ② 平行線ペアによる壁厚自動推定 ③ ドア/窓の高さ分類切り抜き ④ 複数階の自動積み上げ ⑤ 天井スラブ追加。2階建て建物の正確な再現に成功。
v2.0
Zバッファ隠線判定 + 双方向変換の完成
OBJ→DXF変換でVISIBLE比率 0.1% → 78% に改善。法線ベース → Zバッファ深度画像ベースへの手法転換。GPU不使用・numpy のみでバリセントリック座標法のラスタライザを自前実装。

06 — Usage

基本的な使い方

DXF → 3Dアセット（屋根 + テクスチャ付き）

# 建築平面図 → 押し出し + 切妻屋根 + レンガテクスチャ
python cli.py run building.dxf --output-dir ./out --format obj

# TripoSR 外観推論あり（初回 ~2.5GB DL）
python cli.py run building.dxf --output-dir ./out --infer --model triposr

# 陸屋根 + コンクリートテクスチャ
python cli.py run building.dxf --output-dir ./out --roof-type flat --wall-style concrete

3Dモデル → DXF図面

# OBJ → 三面図 + 平面図（Zバッファ隠線判定付き）
python cli.py export model.obj

# 隠線なし（輪郭線のみ）
python cli.py export model.obj --no-hidden

# FBX は事前にOBJへ変換
python fbx2obj.py model.fbx
python cli.py export model.obj
